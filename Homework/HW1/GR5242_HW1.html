def hist_plot(n_sample, n_hist, N_bins, ncol, nrow):
  sample = np.zeros([n_hist, n_sample])
  xlab = "x"
  ylab = "Count"
  fig = plt.figure()
  for i in range(n_hist):
    sample[i, :] = np.random.randn(n)
    ax = fig.add_subplot(ncol,nrow,i+1)
    ax.hist(sample[i,:], bins = N_bins)
    ax.set_xlabel(xlab)
    ax.set_ylabel(ylab)
  plt.suptitle("Sample Size =" + str(n) + ", " + str(N_bins) + " bins")
  plt.show()

e)¶

Generally, the larger the sample size is, the better the result of visualization. As we can see from a) and b), the simulation is unstable when sample size is small, thus affects the reliability of visualization.

In the mean time, the number of bins we select also affect the quality of histograms. When sample size is large and we divide x-axis into only few bins, the estimates become rough and not accurate; when sample size is small but number of bins is large, the histogram becomes several parallel line segments and we can hardly tell what the distribution it is.
